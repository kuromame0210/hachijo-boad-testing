明るい版 UI 仕様書（/bright）

目的:
- 島民向けの運航情報を「少ない情報量・大きめ文字」で一目で見せる
- 「今日動くか」を最優先で表示する

対象ページ:
- `/bright`

情報設計（表示優先順位）:
1. 運航ステータス（運航/欠航/条件付き）
2. 出発/到着時刻
3. 備考（1行）
4. 最終更新時刻

画面構成:
1. ヘッダー
   - 日付（大きめ）
   - 最終更新時刻（右側/小さめ）
   - 更新ボタン（丸ボタン）
2. 交通カード（縦並び）
   - ANA / 東海汽船 / 青ヶ島丸 / ヘリ
   - カード内:
     - ステータスバッジ（色と文言）
     - 出発/到着の時刻（2行）
     - 備考（1行・省略あり）
     - 参照元リンク（小さめ）
3. 天候サマリー
   - 波（底土/八重根）: 波高のみ
   - 風: 平均風速 / 瞬間風速
   - 台風: 「なし」or「接近中」
4. フッター
   - 免責文・更新時刻注記

表示項目（交通）:
- status
- depPlanned
- arrPlanned
- note
- fetchedAt

表示項目（天候）:
- waveHeightM（底土/八重根）
- windSpeedMs / windGustMs
- typhoon有無（台風なしなら「なし」）

デザイン指針:
- 背景: 白〜淡いベージュ
- 文字: 大きめ（見出し 22px、本文 16-18px）
- 色:
  - 運航 = 明るい緑
  - 欠航 = 明るい赤
  - 条件付き = 明るい黄
- 影/枠線は弱めで柔らかく

インタラクション:
- カードタップで詳細（検証ページ）へ
- 更新ボタンは手動更新のみ
